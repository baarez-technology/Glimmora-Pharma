## 10. Glimmora AGI + Autonomous Compliance Layer (Mandatory)

### 10.1 Assisted vs Autonomous Boundaries

| Mode | Description |
|---|---|
| **Assisted AGI** (decision support) | Summarizes, classifies, recommends, drafts, explains with traceability. Requires human review for GxP decisions. |
| **Autonomous AGI** (controlled) | Monitors, flags, prioritizes, orchestrates, escalates based on rules/risk scoring. |
| **Prohibited Autonomy** | Batch disposition/QP release, final QA decisions, CAPA closure without QA approval, external regulator comms, unsupervised learning on production GxP data. |

### 10.2 Autonomous Capabilities
- Compliance Monitoring (CAPA aging, overdue actions, training delinquency, DI exceptions, validation drift)
- Risk Reasoning & Prioritization (ICH Q9-aligned scoring)
- Inspection Readiness Orchestration (evidence kit checks, DIL drills, SME readiness)
- Drift Detection (config changes, access creep, audit trail anomalies)

### 10.3 Guardrails & Controls
- Human-in-the-loop approval gates for regulated decisions
- Explainability (sources, reasoning, confidence, timestamps)
- Immutable audit trail for AGI prompts/outputs/dispositions
- Change control for model/prompt logic updates
- Data protection (redaction, segregation, least privilege, retention)
- Prompt version control, agent action registry, model release governance
- AI qualification package, test harness, regression validation
- Confidence/risk thresholds by use case
- Rollback and containment strategy

### 10.4 AGI Deliverables
- AGI/Autonomy Capability Map
- Intended Use Statements per AGI function
- Risk Classification Matrix
- Human Oversight Model (HITL gates, roles, escalation)
- Monitoring & Drift Concept
- High-level AI validation concept outline

## 23. Architecture & GxP Design Principles

- Human-in-the-loop gates for AI outputs affecting GxP decisions
- Complete traceability and audit trails (who, what, when, why, with what data)
- RBAC enforced at application and data layers
- Evidence kit generation (hashing, versioning, metadata)
- Validation-friendly SDLC and controlled change management for AI
- Digital twin updates traceable to source signals
- Compliance graph relationships must be explainable
- AI agents operate within bounded roles and approval gates
- No autonomous action may bypass regulated human decision rights
- Dashboard metrics must support drill-down to objective evidence
- Risk scoring configurable and ICH Q9-aligned
- Separation between operational systems of record and analytics/AI layers
