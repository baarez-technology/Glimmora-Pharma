# SOW Open Questions

> Questions arising from SOW - GXP-GMP_V3.0 flows and requirements.
> Status: Unanswered — to be reviewed with stakeholders.

---

## Authentication & Multi-Tenancy

1. SSO only or both email+SSO? Will email/password login be supported in production or only during dev/testing?
2. Who manages tenants? Does `super_admin` exist per client org, or is there a Glimmora-level super admin above all tenants?
3. Can one user belong to multiple sites/orgs? How do role assignments work across sites — same role everywhere, or per-site roles?
4. What happens if a user has no site assigned? Blocked at login, or lands on a site picker with empty state?

---

## RBAC & Permissions

5. What is the `viewer` scope — does the viewer see everything read-only, or only specific modules? Can a viewer see AGI policy settings and drift alerts?
6. `operations_head` vs `qa_head` boundary — both have access to the 90-day action plan. Who can edit vs who can only view?
7. Does `qc_lab_director` have any write access to the Batch Readiness screen, or is it read-only for them?
8. Can roles be combined? e.g. can one person be both `qa_head` and `regulatory_affairs`?
9. Who can create/invite users? Only `super_admin`? Can `it_cdo` manage users too?

---

## Dashboard & Readiness Score

10. How is the Readiness Score (0-100) calculated? What is the formula/weighting — is it configurable per site?
11. Who sets the thresholds for Red/Amber/Green on the heatmap? Hard-coded or configurable by `super_admin`?
12. Does the 90-day action plan get created manually or generated by AGI? If AGI-generated, who approves it before it becomes visible?

---

## Gap Assessment & Findings

13. Who can create/enter findings? Only Glimmora consultants during the sprint, or can `qa_head` and `super_admin` add findings from inside the platform?
14. Can findings be deleted or only closed/resolved? Given audit trail requirements, deletion seems non-compliant — how is this handled?
15. Evidence Index — who can upload documents? All roles or restricted?
16. Evidence Pack Status (Complete/Partial/Missing) — is this calculated automatically by the system or manually set?

---

## QMS / CAPA

17. CAPA Source list — is there a fixed enum (483, internal audit, etc.) or free-text? Who controls the list?
18. CAPA closure requires QA approval — does the system enforce a workflow state machine (cannot close without `qa_head` sign-off), or is it advisory?
19. E-signature for CAPA closure — is a full Part 11 e-signature (meaning, intent, timestamp) required at closure, or just an approval click?
20. Effectiveness check scheduling — is this a date-based reminder system, or does the AGI trigger it automatically?
21. Can a CAPA be re-opened after closure? If so, does it create a new record or modify the existing one (immutable audit trail implication)?

---

## CSV/CSA

22. Who enters the system inventory? Manual data entry by `csv_val_lead`, or imported from an existing tool?
23. What are the exact validation status states? (e.g. Validated / Qualified / Not Started / Overdue / Decommissioned)
24. Does the platform itself need to be validated under GAMP 5? Does Glimmora maintain its own validation package for the platform as a GxP system?

---

## AGI / Autonomous Layer

25. Which LLM provider is used? Claude (Anthropic), OpenAI, open source (LLaMA)? On-prem or cloud?
26. Prompt version control — built in-house or using an existing tool (LangSmith, MLflow, etc.)?
27. What triggers a HITL gate? Is the list of HITL-required actions hard-coded, or configurable by `super_admin` via the AGI Policy screen?
28. Confidence threshold values — who sets these and how often can they be changed? Does a change require change control documentation?
29. If the AGI flags something incorrectly (false positive) — what is the dismissal flow? Is the dismissal logged and fed back into the model?
30. Drift detection — what defines "drift"? Is a baseline established at go-live, and how is it updated?

---

## Batch Review & Release

31. Batch Readiness Score — what is the scoring formula and who can configure weights?
32. Can the score be overridden by QA? If QA releases a batch despite a low score, is the override logged with a mandatory reason?
33. Does the platform integrate directly with MES/ERP for batch records, or does someone manually upload them?

---

## Audit Trail

34. Is the audit trail searchable/filterable inside the platform (e.g. "show me all actions by user X on CAPA-042"), or is it export-only?
35. Retention policy — who configures the retention period? What happens when a record reaches end of retention — archive or hard delete?
36. Can `super_admin` view audit logs for all users including other admins? Is there a separate audit log admin role?

---

## Multi-Site / Command Center

37. Is multi-site included in Phase 1 or Phase 2 (follow-on SOW)? The SOW marks most command center features as blueprint-level only.
38. Cross-site data isolation — can Site A users ever see Site B data? Even `super_admin`?

---

## Notifications & Escalations

39. How are alerts delivered? In-app only, or also email / Slack / MS Teams?
40. Who defines escalation rules? `super_admin` in AGI Policy, or is there a separate escalation config?
41. Are there SLA timers on CAPA actions? If so, what happens when one breaches — auto-escalate to whom?

---

## Phase Boundary

42. What is exactly in scope for Phase 1 (this SOW) vs Phase 2? Digital twins, full agent orchestration, command center, and enterprise integrations are all marked "blueprint only" — at what point do they become buildable features?
43. Is the Python AGI layer a separate microservice repo, or does it live inside this Next.js monorepo?

---

## Data Flow & Data Lifecycle

44. How does data enter the system? Manual input forms, file uploads (CSV, PDF, reports), or integrations with external systems (QMS, LIMS, ERP, MES)? Some combination?
45. What is the data lifecycle for compliance records (Findings, CAPAs, Systems, Evidence)? Specifically: can records be edited after creation, or are they immutable with amendment records? What triggers archival vs soft-delete?
46. Is document versioning required? If a user uploads v2 of an SOP, does v1 remain accessible in the audit trail?

---

## API & Backend Contracts

47. Will API endpoint contracts (request/response schemas) be provided by Glimmora, or should the frontend team define them as proposals? If proposals, what format — OpenAPI/Swagger, TypeScript interfaces, or informal?
48. What is the expected API architecture — REST, GraphQL, or RPC? Are there separate services per domain (CAPA service, Findings service) or a monolithic API?
49. For Phase 1, should the frontend use mock service workers (MSW), static JSON fixtures, or Redux-only state for demo purposes?

---

## Workflow & State Machines

50. Beyond CAPA (covered in #17-21), are there formal state machines for other entities? Specifically: Finding lifecycle (Open → In Progress → Closed — can it revert?), System validation status transitions, and FDA 483 event response workflow.
51. Are there SLA-based automatic state transitions? e.g. a Finding that stays "Open" past 30 days auto-escalates to "Overdue", or does this require manual intervention?
52. For approval chains: is it a single-approver model (one QA Head signs off) or can there be sequential multi-approver workflows (e.g. QA Head → Regulatory Affairs → Super Admin)?

---

## UI Behavior & Form Logic

53. What are the required vs optional fields for each entity form (CAPA, Finding, System, Evidence, FDA 483 event)? The current UI has form fields but no documented validation schema per entity.
54. Should forms auto-save as drafts, or only persist on explicit "Save" action? For long forms (e.g. CAPA creation with RCA details), losing work on accidental navigation would be a poor UX.
55. What is the expected behavior for concurrent edits? If two users open the same CAPA simultaneously, does last-write-win, or is there optimistic locking / conflict resolution?

---

## Scope & MVP Prioritization

56. What is the priority order for building the remaining incomplete modules? Specifically: Dashboard, Inspection Readiness, and Governance are currently placeholders. Which should be completed first?
57. Is this phase purely a UI/demo deliverable (all mock data, no persistence required), a functional prototype (localStorage persistence, realistic workflows), or a production-ready frontend (API-integrated, fully tested)?
58. Are there specific demo scenarios or walkthroughs that stakeholders want to present? This would help prioritize which mock data and user flows to polish first.

---

## Error Handling & Edge Cases

59. What should happen when a user attempts an action they don't have permission for? Silent hide (button not shown), disabled state with tooltip, or redirect to an "Access Denied" page?
60. How should the UI handle API failures once connected? Retry with exponential backoff, show inline error with retry button, or redirect to a generic error page?
61. What is the expected behavior for empty states (e.g. new site with zero CAPAs, zero Findings)? Should the system show onboarding prompts, sample data, or just an empty table with a CTA?
